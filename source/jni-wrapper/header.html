jni-wrapper

Building this library

The purpose of each target defined in the Makefile for this directory is listed below. The files are written to or removed from their respective output directory as follows:
  • Object (.o) files - .obj
  • Header (.h) files - $JASL_BASE/include/jasl/jni
  • Library files and links - $JASL_BASE/lib/jasl
  • obj - compiles all the source files using gcc, placing the object (.o) files in the directory noted above.
  • hdr - copies the header (.h) files to the directory noted above.
  • none or libs - executes the actions defined for the obj target and then uses the object (.o) files to generate shared and static libraries, as well as the necessary links, in the directory noted above. Upon completion, the clean target is used to remove the directory containing the object (.o) files.

  • clean - removes the directory noted above that is used to store the object (.o) files.
  • clean_hdr - deletes the copies of the header (.h) files in the directory noted above.
  • clean_libs - deletes the shared and static library files, as well as the associated links, in the directory noted above.

    The following targets are also referenced by the Makefile in the parent directory. The output files are placed in the indicated subdirectory of $JASL_BASE/share/jasl/doc

  • docs - generates an HTML version of each source file and the Makefile using source-highlight, placing the output files in source/jni-wrapper. This file, jni-wrapper.html, is also generated, and includes links to all of the HTML files.
  • doxygen - generates class documentation for the JNI wrapper libraries using doxygen, placing the output files in jni-doxygen/html.
