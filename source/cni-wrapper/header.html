cni-wrapper

Building this library

The purpose of each target defined in the Makefile for this directory is listed below. The header and library files are written to or removed from $JASL_BASE/include and $JASL_BASE/lib, respectively. Note that jaslWrapper.h is not directly associated with this library. It provides typedef declarations to simplify access to classes in the jasl libraries through programs that require it for execution. See Driver.cpp for an example.

  • none or obj - uses GCC to compile the source (.cpp) file into an object (.o) file.
  • hdr - copies the header (.h) files to the directory noted above.
  • libs - executes the actions defined for the obj target and then uses the object (.o) file to generate shared and static libraries, as well as the necessary links, in the directory noted above. Upon completion, the clean target is used to remove the object (.o) file.

  • clean - deletes the object (.o) file in this directory.

  • clean_hdr - deletes the copies of the header (.h) files in the directory noted above.

  • clean_libs - executes the actions defined for the clean_dynamic_lib and clean_static_lib targets.
  • clean_dynamic_lib - deletes the shared library and its associated links in the directory noted above.
  • clean_static_lib - deletes the static library file in the directory noted above.

  • docs - generates an HTML version of each source file and the Makefile using source-highlight, placing the output files in $JASL_BASE/share/jasl/doc/source/cni-wrapper. This file, cni-wrapper.html, is also generated, and includes links to all of the HTML files. This target is also referenced by the Makefile in the parent directory.
