jasl

Building the packages defined in the sub-directories

The purpose of each target defined in the Makefile for this directory is listed below. The files are written to or removed from their respective output directory as follows:
  • Bytecode (.class) - $JASL_BASE/bin/jasl/< sub-directory name >
  • Object (.o) files -
  • Header (.h) files - $JASL_BASE/include/jasl/< sub-directory name >
  • Library files and links - $JASL_BASE/lib
Note that the majority of the targets are also referenced by the Makefile in the parent directory during recursive cleans and builds. Nearly all of them "pass through" to the sub-directories, where the actual execution takes place.
  • none or all - compiles all the source files in each sub-directory using javac, placing the bytecode (.class) files in the corresponding directory noted above.
  • obj - compiles all the bytecode (.class) files generated in each sub-directory using gcj, placing the object (.o) files in the corresponding directory noted above.
  • hdr - generates a CNI header file from each bytecode (.class) file using gjavah, placing the header (.h) files in the corresponding directory noted above.
  • libs - executes the actions defined for the obj target and then uses the object (.o) files to generate dynamic and static libraries, as well as the necessary links, in the directory noted above. Upon completion, the clean_obj target is used to remove the object (.o) files.

  • clean - executes the actions defined for the clean_java and clean_obj targets.
  • clean_java - deletes all of the bytecode (.class) files in the directories noted above.
  • clean_obj - deletes all of the object (.o) files in the directories noted above.

  • clean_hdr - deletes all of the header (.h) files in the directories noted above.

  • clean_libs - executes the actions defined for the clean_dynamic_libs and clean_static_libs targets.
  • clean_dynamic_libs - deletes the dynamic library files and their associated links in the directory noted above.
  • clean_static_libs - deletes the static library files in the directory noted above.

  • docs - generates an HTML version of the Makefile using source-highlight, placing it in $JASL_BASE/share/jasl/doc/source/jasl. This file, jasl.html, is also generated, and includes links to all of the HTML files generated by the same target, called from this one, in each sub-directory.
