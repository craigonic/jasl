jasl.utilities

Building this package

The purpose of each target defined in the Makefile for this directory is listed below. The files are written to or removed from their respective output directory as follows:
  • Bytecode (.class) and object (.o) files - $JASL_BASE/bin/jasl/utilities
  • Header (.h) files - $JASL_BASE/include/jasl/utilities
  • Library files and links - $JASL_BASE/lib
Note that many of the targets are also referenced by the Makefile in the parent directory during recursive cleans and builds.
  • none or all - compiles all the source files using javac, placing the bytecode (.class) files in the directory noted above.
  • obj - compiles all the source files using gcj, placing the object (.o) files in the directory noted above.
    This target is also used in builds initiated from the top-level directory. The bytecode (.class) files for each source file are also generated and placed in the appropriate location as a result of its execution.
  • hdr - generates a CNI header file from each bytecode (.class) file using gjavah, placing the header (.h) files in the directory noted above.
  • libs - executes the actions defined for the obj target and then uses the object (.o) files to generate dynamic and static libraries, as well as the necessary links, in the directory noted above. Upon completion, the clean_obj target is used to remove the object (.o) files.

  • filename.class - compiles filename.java into a bytecode file using javac, placing filename.class in the directory noted above.
  • filename.o - compiles filename.java into an object file using gcj, placing filename.o in the directory noted above.
  • filename.h - generates a CNI header file from filename.class using gjavah, placing the output file in the directory noted above.

  • clean - executes the actions defined for the clean_java and clean_obj targets.
  • clean_java - deletes all of the bytecode (.class) files in the directory noted above.
  • clean_obj - deletes all of the object (.o) files in the directory noted above.

  • clean_hdr - deletes all of the header (.h) files in the directory noted above.

  • clean_libs - executes the actions defined for the clean_dynamic_lib and clean_static_lib targets.
  • clean_dynamic_lib - deletes the dynamic library and its associated links in the directory noted above.
  • clean_static_lib - deletes the static library file in the directory noted above.

  • docs - generates an HTML version of each source file and the Makefile using source-highlight, placing the output files in $JASL_BASE/share/jasl/doc/source/jasl/utilities. This file, utilities.html, is also generated, and includes links to all of the HTML files.
