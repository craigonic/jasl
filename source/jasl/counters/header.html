Counters package

Generation of all the files and documentation in the Counters package relies heavily on the 'make' program. It is not absolutely necessary, particularly for building the .class files, but it does simplify the process. For those who might question this approach, I want to assure you that I am aware of alternatives, such as Ant. For now, I am using the tools I am most familiar with.

The following conditions are assumed about the system used to process the files in this directory.

  • The original directory structure for this system is in place. The top level directory is jasl, which is referred to as ROOTDIR in this document.
  • Java SE SDK (1.2 or higher). I use the Blackdown port for Linux, but the Sun versions should also work.
  • Optional. make.
  • Optional. GCJ (3.0 or higher). This can be used to build a native executable version of this package.
  • Optional. src-highlite. This program is used to create HTML versions of the source files.
  • Optional. Perl and sed, which are used to build a linked hierarchy of the HTML versions of the source files.
  • Optional. A toolchain capable of building an executable with gcj for a non-iX86 architecture. I have used these source files and build structure, with only the minor change noted below for the gcj and gcj_obj targets, to cross-compile an executable and run it on my Sharp Zaurus.

The purpose of each target defined in the Makefile is listed below. The executable and object files are written to (and deleted from) the output directory - ROOTDIR/jasl/Counters. Note that most of the targets are also used by the Makefile in the parent directory during recursive cleans and builds.

  • none or all or build - compiles all the source files using javac, placing the .class files in the output directory.
  • gcj or gcj_obj - compiles all the source files using gcj, placing the .o files in the output directory. To specify an alternative gcj compiler, such as arm-linux-gcj, specify the execution of make as follows: make GCJ_COMPILER=arm-linux-gcj gcj

  • filename.class - compiles filename.java into a bytecode file using javac, placing filename.class in the output directory.
  • filename.o - compiles filename.java into an object file using gcj, placing filename.o in the output directory.

  • clean_java - deletes all of the .class files in the output directory.
  • clean_obj - deletes all of the .o files in the output directory.
  • clean - executes the actions defined for the clean_java and clean_obj targets.

  • docs - generates an HTML version of each source file, placing the output files in ROOTDIR/docs/source/Counters. This file, ROOTDIR/docs/source/Counters/Counters.htm, is also generated, and includes links to all of the HTML files.
  • javadocs - generates class documentation for this package using the javadoc program. The output files are placed in ROOTDIR/docs/javadocs.
