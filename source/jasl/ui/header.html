jasl.ui

Building the packages defined in the sub-directories

The purpose of each target defined in the Makefile for this directory is listed below. The files are written to or removed from their respective output directory as follows:
  • Bytecode (.class) files - $JASL_BASE/bin/jasl/ui/< package name >
  • Object (.o) files - < package name >/.obj
  • Header (.h) files - $JASL_BASE/include/jasl/< package name >
  • Library files and links - $JASL_BASE/lib/jasl
Note that the majority of the targets are also referenced by the Makefile in the parent directory during recursive cleans and builds. Nearly all of them "pass through" to the sub-directories, where the actual execution takes place.
  • none or all - compiles all the source files in each sub-directory using javac, placing the bytecode (.class) files in the corresponding directory noted above.
  • obj - compiles all the bytecode (.class) files generated in each sub-directory using gcj, placing the object (.o) files in the corresponding directory noted above.
  • hdr - generates a CNI header (.h) file from each bytecode (.class) file using gjavah, placing the output files in the corresponding directory noted above.
  • libs - executes the actions defined for the obj target and then uses the object (.o) files to generate shared and static libraries, as well as the necessary links, in the directory noted above. Upon completion, the clean_obj target is used to remove the directories containing the object (.o) files.

  • clean - executes the actions defined for the clean_java and clean_obj targets.
  • clean_java - deletes the bytecode (.class) files in each of the directories noted above.
  • clean_obj - removes each of the directories noted above that contain object (.o) files.

  • clean_hdr - deletes the generated header (.h) files in each of the directories noted above.
  • clean_libs - deletes the shared and static library files, as well as the associated links, in the directory noted above.

  • docs - generates an HTML version of the Makefile using source-highlight, placing it in $JASL_BASE/share/jasl/doc/source/jasl/ui. This file, ui.html, is also generated, and includes links to all of the HTML files generated by the same target, called from this one, in each sub-directory.
