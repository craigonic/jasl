org.json

Building this package

The purpose of each target defined in the Makefile for this directory is listed below. The files are written to or removed from their respective output directory as follows:
  • Bytecode (.class) files - $JASL_BASE/bin/org/json
  • Object (.o) files - .obj
  • Header (.h) files - $JASL_BASE/include/org/json
  • Library files and links - $JASL_BASE/lib/jasl
  • none or all - compiles selected source files in the json sub-directory using javac, placing the bytecode (.class) files in the directory noted above.
  • jar - create an executable jar archive containing the bytecode (.class) files generated with the all target.
  • obj - compiles bytecode (.class) files using gcj, placing the object (.o) files in the directory noted above.
  • hdr - generates a CNI header (.h) file from each bytecode (.class) file using gjavah, placing the output files in the directory noted above.
  • libs - executes the actions defined for the obj target and then uses the object (.o) files to generate shared and static libraries, as well as the necessary links, in the directory noted above. Upon completion, the clean_obj target is used to remove the directory containing the object (.o) files.

  • clean - executes the actions defined for the clean_java and clean_obj targets.
  • clean_java - deletes the bytecode (.class) files in the directory noted above.
  • clean_obj - removes the directory noted above that contains the object (.o) files.

  • clean_jar - removes the jar archive.
  • clean_hdr - deletes the generated header (.h) files in the directory noted above.
  • clean_libs - deletes the shared and static library files, as well as the associated links, in the directory noted above.

  • docs - generates an HTML version of the Makefile using source-highlight, placing the output file in $JASL_BASE/share/jasl/doc/source/org. This file, org.html, is also generated, and includes links to all of the HTML files.

    The following targets are used to get and update a reference implementation of a JSON package in Java. It is managed as a git submodule, "cloned" into the json sub-directory.

  • clone - create a local copy of the source repository.
  • refresh - get the latest updates.
