swig

Building the wrapper libraries and their associated modules

The purpose of each target defined in the Makefile for this directory is listed below. The output files are written to or removed from $JASL_BASE/lib/{scripting language name (lower case)} for the libraries or the sub-directories of $JASL_BASE/bin, associated with their scripting language below, for the module files.

  • perl - perljASL
  • python - pyjASL

    The following targets are used to build and clean the libraries and module files necessary to access the CniWrapper library using the scripting language name (perl or python) included in the target name. The desired version(s) of this library and its associated files must be available in order for the other wrapper libraries to work.

  • cni_{language name}_src - uses swig to generate the library source file (CniWrapper_wrap.cxx). If successful, it also creates the module file, placing it in the directory noted above.

  • cni_{language name}_obj - executes the actions defined for the cni_{language name}_src target and then uses gcc to compile the generated source file into an object (.o) file.

  • cni_libs - executes the actions defined for each cni_{language name}_lib target.

  • cni_{language name}_lib - executes the actions defined for the cni_{language name}_obj target and then uses the object (.o) file to generate a shared library (CniWrapper.so) in the directory noted above.

  • clean_cni - executes the actions defined for the clean_cni_src and clean_cni_obj targets. Note that these targets work for all scripting languages.

  • clean_cni_src - deletes the library source file (CniWrapper_wrap.cxx).
  • clean_cni_obj - deletes the library object file (CniWrapper_wrap.o).

  • clean_cni_libs - executes the actions defined for the clean_cni_{language name} targets.

  • clean_cni_{language name} - executes the actions defined for the clean_cni_{language name}_module and clean_cni_{language name}_lib targets.

  • clean_cni_{language name}_module - deletes the module file(s) from the directory noted above.
  • clean_cni_{language name}_lib - deletes the library from the directory noted above.

    The following targets are used to build and clean all of the libraries and files referenced above, using combinations of the targets associated with them, and the indicated documentation.

  • none or libs - executes the actions defined for each {library name}_libs target, followed by the clean target.

  • clean_all - executes the actions defined for the clean and clean_libs targets.

  • clean - executes the actions defined for each clean_{library name} target.
  • clean_libs - executes the actions defined for each clean_{library name}_libs target.

  • docs - generates an HTML version of each interface file and the Makefile using source-highlight, placing the output files in $JASL_BASE/share/jasl/doc/source/swig. This file, swig.html, is also generated, and includes links to all of the HTML files. This target is also referenced by the Makefile in the parent directory.
