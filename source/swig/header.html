SWIG

Building the wrapper libraries and their associated modules

The purpose of each target defined in the Makefile for this directory is listed below. The library and module files are written to or removed from $JASL_BASE /lib/{scripting language name (lower case)} .

    The following targets are used to build and clean the libraries and module files necessary to access the CniWrapper library using the scripting language name (perl or python) included in the target name. The desired version(s) of this library and its associated files must be available in order for the other wrapper libraries to work.

  • cni_{language name}_src - uses SWIG to generate the library source file (CniWrapper_wrap.cxx). If successful, it also creates the module file, placing it in the output directory.

  • cni_{language name}_obj - executes the actions defined for the cni_{language name}_src target and then uses GCC to compile the generated source file into an object (.o) file.

  • cni_libs - executes the actions defined for each cni_{language name}_lib target.

  • cni_{language name}_lib - executes the actions defined for the cni_{language name}_obj target and then uses the object (.o) file to generate a shared library (CniWrapper.so) in the directory noted above.

  • clean_cni - executes the actions defined for the clean_cni_src and clean_cni_obj targets. Note that these targets work for all scripting languages.

  • clean_cni_src - deletes the library source file (CniWrapper_wrap.cxx).
  • clean_cni_obj - deletes the library object file (CniWrapper_wrap.o).

  • clean_cni_libs - executes the actions defined for the clean_cni_{language name} targets.

  • clean_cni_{language name} - executes the actions defined for the clean_cni_{language name}_module and clean_cni_{language name}_lib targets.

  • clean_cni_{language name}_module - deletes the module file(s) from the output directory.
  • clean_cni_{language name}_lib - deletes the library from the output directory.

    The following targets are used to build and clean the libraries and module files necessary to access the Counters library using the scripting language name (perl or python) included in the target name.

  • counters_{language name}_src - uses SWIG to generate the library source file (Counters_wrap.cxx). If successful, it also creates the module file, placing it in the output directory.

  • counters_{language name}_obj - executes the actions defined for the counters_{language name}_src target and then uses GCC to compile the generated source file into an object (.o) file.

  • counters_libs - executes the actions defined for each counters_{language name}_lib target.

  • counters_{language name}_lib - executes the actions defined for the counters_{language name}_obj target and then uses the object (.o) file to generate a shared library (Counters.so) in the directory noted above.

  • clean_counters - executes the actions defined for the clean_counters_src and clean_counters_obj targets. Note that these targets work for all scripting languages.

  • clean_counters_src - deletes the library source file (Counters_wrap.cxx).
  • clean_counters_obj - deletes the library object file (Counters_wrap.o).

  • clean_counters_libs - executes the actions defined for the clean_counters_{language name} targets.

  • clean_counters_{language name} - executes the actions defined for the clean_counters_{language name}_module and clean_counters_{language name}_lib targets.

  • clean_counters_{language name}_module - deletes the module file(s) from the output directory.
  • clean_counters_{language name}_lib - deletes the library from the output directory.

    The following targets are used to build and clean all of the libraries and files referenced above, using combinations of the targets associated with them, and the indicated documentation.

  • libs - executes the actions defined for each {library name}_libs target, followed by the clean target.

  • clean_all - executes the actions defined for the clean and clean_libs targets.

  • clean - executes the actions defined for each clean_{library name} target.
  • clean_libs - executes the actions defined for each clean_{library name}_libs target.

  • docs - generates an HTML version of each interface file and the Makefile using source-highlight , placing the output files in $JASL_BASE/share/jasl/doc/source/swig. This file, swig.html, is also generated, and includes links to all of the HTML files. This target is also referenced by the Makefile in the parent directory.
